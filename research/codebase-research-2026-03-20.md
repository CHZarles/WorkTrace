# RecorderPhone Codebase Research

## High-Level Summary
The repository currently combines a Rust Core service, a Windows collector, a Manifest V3 browser extension, multiple Flutter clients, an Android UsageStats plugin, and separate Windows and Android app trees. The Rust Core is the central runtime: it owns the HTTP API, SQLite schema, privacy handling, tracking pause state, block and timeline aggregation, exports, and report generation/scheduling. The Windows collector and browser extension both send ingest events into the Core’s `/event` endpoint, while the Flutter desktop app talks back to the Core over HTTP for review, settings, reporting, and export workflows. The Flutter Android path is separate from the Core-backed desktop path: it reads UsageStats through a Flutter plugin, stores generated blocks locally in SQLite, and renders its own Today, Review, and Settings screens. The repository also contains a separate Flutter app under `recorderphone_ui`, a WinUI 3 Windows app with a local ingest server, and an Android Compose app with a static Today screen.

## Rust Core Service
- The Core process initializes tracing, parses CLI arguments, creates the database directory, opens SQLite, loads app and report settings, builds a CORS-enabled Axum router, and starts a background report scheduler before serving HTTP. `core/recorder_core/src/main.rs:936-1080`
- The router exposes health, ingest, recent events, current-state, tracking pause/resume, settings, timeline, block review/delete, privacy rules, day delete/wipe, markdown/csv export, report settings, daily/weekly generation, report TODOs, and report list/detail routes. `core/recorder_core/src/main.rs:984-1065`
- Event ingest accepts raw JSON, deserializes into `IngestEvent`, validates version and RFC3339 timestamps, requires `domain` for `tab_active`/`tab_audio_stop` and `app` for `app_active`/`app_audio`/`app_audio_stop`, checks tracking pause state, applies privacy rules and global storage settings, and persists the final JSON payload into SQLite. `core/recorder_core/src/main.rs:1154-1352`
- The `/now` snapshot is derived from recent `app_active`, `tab_active`, `tab_audio_stop`, `app_audio`, and `app_audio_stop` rows, filtered through the privacy index and evaluated with separate focus/audio TTLs plus latest-title hints. `core/recorder_core/src/main.rs:358-683`
- Block aggregation splits focus events from background-audio events, resolves browser app focus into either app or domain attribution based on domain freshness, truncates segments at idle cutoffs, and can attach background audio into block summaries. `core/recorder_core/src/main.rs:6350-6878`
- Timeline generation builds foreground focus segments and background audio segments separately, then merges them into the response model with `kind`, `entity`, `title`, `activity`, `start_ts`, `end_ts`, and `seconds`. `core/recorder_core/src/main.rs:6590-6770`
- SQLite initialization creates and maintains the tables that back events, block reviews, app settings, privacy rules, tracking state, report settings, reports, and report TODO data. `core/recorder_core/src/main.rs:5104-5668`
- Report generation uses an OpenAI-compatible `/chat/completions` request, extracts markdown from the first choice, sanitizes the output, constructs daily or weekly report inputs from blocks, timeline segments, settings, privacy rules, and TODOs, stores report rows, and can write markdown/csv files under the data directory. `core/recorder_core/src/main.rs:3959-4035` `core/recorder_core/src/main.rs:4184-4995`
- A scheduler loop runs every 30 seconds and auto-generates daily and weekly reports when report settings are configured and the scheduled local time has passed, using cooldown tracking per report id. `core/recorder_core/src/main.rs:4997-5102`

## Ingest Producers

### Windows Collector
- The Windows collector exposes CLI options for the Core URL, poll interval, title/exe-path capture, heartbeat, background-audio tracking, review notifications, repeat intervals, and idle cutoff. The Windows entrypoint enforces a single-instance mutex, posts `app_active` and `app_audio`/`app_audio_stop` events to `/event`, and resets foreground attribution when system idle time exceeds the configured cutoff. `collectors/windows_collector/src/main.rs:1-170` `collectors/windows_collector/src/main.rs:332-365`
- Foreground detection reads the active window handle, pid, title, and executable path through Win32 APIs; background-audio detection enumerates CoreAudio sessions, filters out browser executables and `audiodg.exe`, and reports the selected process as an audio source. `collectors/windows_collector/src/main.rs:628-805`
- Review notification polling calls the Core’s `/tracking/status` and `/blocks/due` endpoints, formats the due block time range and top items, and emits Windows toast actions that deep-link into `recorderphone://review`, `recorderphone://review?action=skip`, and `recorderphone://review?action=pause&minutes=15`. `collectors/windows_collector/src/main.rs:368-598`

### Browser Extension
- The MV3 extension declares `tabs`, `storage`, `alarms`, and `offscreen` permissions, allows localhost HTTP targets, runs a module service worker, and exposes a popup UI. `extension/manifest.json:1-16`
- The service worker keeps defaults and mutable state in Chrome storage, restores prior state from `chrome.storage.local`, tracks offscreen-document status, posts events to `/event`, and re-establishes a one-minute heartbeat alarm during install, startup, message handling, and worker bootstrap. `extension/service_worker.js:1-260` `extension/service_worker.js:420-587`
- Active-tab emission distinguishes focused browsing from audible background tabs: when the browser is focused it emits `tab_active` with `activity: "focus"` for the active tab in the last focused window; when the browser is not focused and background-audio tracking is enabled it emits `tab_active` with `activity: "audio"` for an audible tab, and emits `tab_audio_stop` when audio ends or focus returns to the browser. `extension/service_worker.js:270-408`
- Runtime message handlers support keep-alive pings from the offscreen document plus `diagnose`, `repair`, and `forceEmit` control messages. `extension/service_worker.js:490-576`
- The popup loads synced settings, renders status from `chrome.storage.local`, and exposes controls for enable/disable, tab title sending, background-audio tracking, keep-alive, server URL, `/health` testing, force send, and repair. `extension/popup.js:1-158` `extension/popup.html:1-64`
- The offscreen document exists only to periodically send `keepAlivePing` messages back to the service worker. `extension/offscreen.js:1-17`
- The shared JSON schema defines the accepted ingest event shapes for the browser extension and Windows collector, including `tab_active`, `tab_audio_stop`, `app_active`, `app_audio`, and `app_audio_stop`. `schemas/ingest-event.schema.json:1-127`
